# TXL-PBC 血液抹片細胞偵測報告

> 影像處理 114-2 ｜ 期末專案書面報告
> 一個**不使用深度學習、也不使用任何訓練模型**的血液抹片細胞計數 Streamlit App。

本報告與目前程式碼（`blood_cell_detector.py`、`app.py`、`eval_iou.py`）同步，
所有數據皆由 `eval_iou.py` 依助教指定的 **IoU 評估法**實際執行重現。

## 一、設計總覽與「為什麼這樣做」

整體策略：**先理解每種細胞在影像上的可分辨特徵，再用最少、最可解釋的規則把它框出來**。
三種細胞的核心判別依據各不相同，這是刻意的設計選擇：

| 細胞 | 最可靠的判別特徵 | 為什麼選這個 |
|------|------------------|--------------|
| WBC | **真紫色（violet）細胞核 + 深染核心** | 白血球核染成真正的紫羅蘭色（hue 120–158），RBC 即使成團也偏粉紅；用「窄 hue + 寬鬆飽和/亮度」可連淡染核都抓到；再加「核心須深染」驗證（飽和度 ≥ 100 或 LAB-B ≤ 102），且**過大候選須核心飽和度明顯高出整張影像**，即使整張抹片偏紫也不會把紫色染色假象框成橫跨全圖的白血球 |
| RBC | **形狀 + 解析度推算的半徑（+ Hough 圓補強）** | RBC 數量多、顏色淡、彼此重疊，用顏色不可靠；但大小極一致，適合幾何規則；相連／環不完整的細胞再用 Hough 圓偵測（固定半徑先驗）補回 |
| 血小板 | **紫色 + 顆粒狀內部紋理** | 血小板小且色近 WBC 碎片，單靠顏色/大小會誤抓雜訊；其內部顆粒造成的「亮度變異」才是關鍵 |

兩個貫穿全專案的設計：
1. **所有尺寸門檻都以「由影像解析度估計的 RBC 半徑 `r0`」為基準縮放**（`shape_rbc_radius()`），不偷看標註答案，讓同一組固定參數能在不同解析度影像上通用。
2. **先偵測 WBC、再偵測其他**：WBC 框先建立後，RBC 與血小板會排除「中心落在 WBC 框內」的候選，避免一顆白血球被切成好幾顆紅血球或血小板。

## 二、Pipeline 架構

```
輸入 RGB 影像
    ├─ 灰階 Grayscale
    ├─ 高斯模糊 Gaussian Blur
    ├─ 自適應二值化 Adaptive Thresholding
    ├─ 形態學閉運算 + 開運算 Closing & Opening
    ├─ 輪廓抽取 Contour extraction
    ├─ 面積 / 圓形度 / 長寬比 / 顏色 / 紋理 規則
    ├─（選用）Distance Transform + Watershed（重疊 RBC）
    └─ 計數 WBC / RBC / Platelet 並繪製方框
```

## 三、三種細胞偵測：方法與設計決策

### WBC（`detect_wbc`，最先執行）

**做法**：**violet-hue 遮罩**（`wbc_violet_mask`：hue 120–158、S≥70、V<232、A>132、B<128）→
形態學**開運算**取出實心核 → 小幅膨脹合併同一核的分葉 → 連通元件 → 幾何（夠大）規則 +
核心深染驗證 + **過大候選的相對飽和度閘**（邊長 ≥ 4.5·r0 者須 `s_core − 影像 S 中位數` ≥ 70，
擋掉整張偏紫的染色假象，見第四節「修正 7」）→ 中心距離 NMS。框取核心外接框（GT ≈ 1.03×核外接框，只加極小 padding）。

**為什麼用窄 hue 而非高飽和**：見第四節「困難案例 1、4」。RBC 即使成團仍偏粉紅，WBC 核是
真紫色；把 hue 鎖在 120–158 就能把 RBC 團擋掉，於是飽和度/亮度門檻可以放寬，連淡染
（低飽和、偏亮）的白血球核也抓得到。

**為什麼先偵測 WBC**：偵測完的 WBC 框會傳給 RBC 與血小板偵測器，凡中心落在 WBC 框內的
候選一律排除，白血球就不會被誤切成紅血球或血小板。

### RBC（`detect_rbc_rule`）

**做法**：灰階 → 高斯模糊 → 自適應二值化 → 閉/開運算 → `findContours`（`RETR_LIST`）→
面積、圓形度、長寬比規則篩選，並排除落在紫色區內的輪廓（避免把 WBC 當 RBC）→
**包含抑制**（去除包住單顆細胞的過大團塊框，見第四節「修正 6」）→
**Hough 圓偵測補強**（補回相連／環不完整、輪廓抓不到的紅血球，見第四節「修正 11」）。

**為什麼用幾何而非顏色**：RBC 顏色淡、H 值雙峰、又常重疊，顏色極不穩定；
但 RBC 大小非常一致（GT 框邊長 ≈ 2·r0，IQR 僅 1.79–2.15），所以**用解析度推得的 `r0`
反推框大小**比量測輪廓更準（見第四節「困難案例 2」）。

### 血小板（`detect_platelets_rule`）

**做法**：寬鬆紫色遮罩取小連通元件 → 抽取手工特徵 → 用**飽和度變異 `S_std` 與灰階變異
`gray_std`**（顆粒紋理）配合顏色（`S_mean`、`B_mean`）與面積門檻篩選 →
排除中心落在**加 margin（0.22·r0）的 WBC 框**內的候選（去掉白血球邊緣的假血小板，見第四節「修正 8」）→ 固定框大小。

**為什麼用「紋理變異」當主判據**：見第四節「困難案例 3」。真正的血小板內部有顆粒，
造成明顯的亮度/飽和度起伏；而背景的紫色雜訊是均勻的。在訓練集上量測發現，
血小板的 `S_std` 中位數約 39、雜訊僅約 2.4——這是最乾淨的分界，遠比面積/圓形度有效。

## 四、優化歷程（為什麼指標會這樣演進）

### 起點：評估方法本身就錯了

最初的評估用「預測框中心是否落在 GT 框內」來判定 TP（center-in matching）。
這太寬鬆——只要框在大概位置就算對，**框大小完全不檢查**。在這個標準下三類細胞 F1 都
很漂亮（WBC 0.85、RBC 0.72、PLT 0.91），卻**完全沒反映框得準不準**。

### 改用助教指定的 IoU 評估法（`eval_iou.py`）

依公告實作：YOLO 標註 → 像素框（含邊界裁切）→ `IoU = 交集/聯集` → 一對一貪婪配對
（同類別、IoU 超過閾值才算 TP）。換成 IoU 後，真實表現暴露出來（test, IoU 0.5）：

| 細胞 | center-in（舊） | IoU 0.5（真實） |
|------|----------------|-----------------|
| WBC | 0.85 | **0.22** |
| RBC | 0.72 | **0.51** |
| PLT | 0.91 | 0.77 |

WBC 從 0.85 崩到 0.22，證明框嚴重失準。**這個落差正是改用 IoU 才看得到的——
也呼應公告所說「若僅以數字分數分辨，無法得知是否因誤差而辨識成功」。**

### 修正 1：WBC 框爆成整張圖

診斷第一張 test 圖：WBC 預測框是 `[0,0,575,542]`（整張圖），GT 只有 `[170,176,395,417]`。
原因：紫色遮罩在這張圖抓到 ~24% 的像素（連 RBC 邊緣淡紫都中），21px 膨脹把所有零散
紫色併成一個橫跨全圖的連通塊。
**解法**：先用開運算（kernel≈0.18·r0）只留實心核，再小幅膨脹（0.20·r0）只合併同核分葉。
量測後得知 GT 框 ≈ 1.03×核外接框，故只加極小 padding。
→ WBC F1（IoU 0.5）**0.22 → 0.53**。

### 修正 2：RBC 框普遍太小

量測發現預測 RBC 框邊長中位數 1.67·r0，但 GT 是 1.99·r0——框系統性偏小，IoU 因此不足。
**解法**：RBC 大小極一致，直接把框半徑往 `r0` 收斂（clip 到 [0.88, 1.18]·r0）。
→ RBC F1（IoU 0.5）**0.51 → 0.67**，micro **0.49 → 0.67**。

### 修正 3：規則式血小板誤判爆量

純規則血小板一開始預測了 1634 個（GT 僅 49，超量 33 倍），F1 僅 **0.02**。
原因：寬鬆紫色遮罩 + 寬鬆面積門檻把所有小紫點都當血小板。
**解法**：在訓練集上比較「真血小板 vs 雜訊」的特徵分布，發現**顆粒紋理**（`S_std`、`gray_std`）
是最乾淨的分界，加上顏色與面積門檻，並把框設成固定大小（≈0.84·r0）以對齊 GT。
→ 雜訊預測 1617 → 21，PLT F1（IoU 0.3）**0.02 → 0.68**（val 0.75）。

**這個修正的副作用是好的**：血小板不再需要先前訓練的 ExtraTrees 模型，
整條 pipeline 變成純規則、零模型依賴，Colab notebook 因此能完全自包含、無需下載 29MB 模型。

### 修正 4：淡染白血球漏抓 + 白血球被切成紅血球

診斷 test 圖 `13cd097a…`：一顆白血球**完全沒被偵測到**，反而被 RBC 偵測器切成 3 顆紅血球。
原因：這張圖的白血球核是**淡染**的（飽和度僅 ~30、偏亮），原本「高飽和」的紫色遮罩抓不到，
於是 `detect_wbc` 回傳空、該區域被 RBC 接管。
**解法（兩部分）**：
1. **改用 violet-hue 遮罩**：量測發現白血球核是真紫色（hue≈141），RBC 團是粉紅；把 hue
   鎖在 120–158、同時放寬飽和度/亮度，淡染核也抓得到，又不會誤收 RBC 團。
   （試過 CLAHE 增強對比，但它同時放大雜訊飽和度、反而增加誤判，故不採用。）
2. **先偵測 WBC、再排除**：把 WBC 框傳給 RBC/血小板偵測器，排除中心落在 WBC 框內的候選。
→ WBC F1（test IoU 0.3）**0.70 → 0.89**（召回率 0.68 → 0.95），且白血球不再被切成紅血球；
血小板因排除 WBC 內誤判，精確率亦從 0.63 升到 0.74。

### 修正 5：整張偏紫的抹片——淡紫紅血球被誤判 + 漏抓

診斷 test 第一張圖 `00552f60…`：左側一顆紅血球被當成白血球（多框一顆 WBC），
右側幾顆明顯的紅血球卻完全沒被偵測到（RBC 只抓到 4 / 9）。
**原因**：這張抹片整體**偏紫**，紅血球本身染得偏紫紅（hue≈160、B≈117–129），
於是同時觸發兩個問題：
1. **被當白血球**——淡紫紅血球團通過了 violet-hue 遮罩，`detect_wbc` 多框一顆。
2. **被漏抓**——`purple_mask_strict`（供 RBC 偵測「排除紫色區」用）把這些淡紫紅血球也當紫色排除掉，整顆紅血球因此消失。

**關鍵觀察**：對全資料集量測「核心區」的特徵分布後發現，
真白血球核**染色極深**（核心飽和度中位數 178、99% ≥ 95；LAB 之 B 中位數 87），
而淡紫紅血球團即使偏紫，核心仍淡（飽和度 ~83、B ~117）；
LAB 的 **B 通道**最能分界（白血球 B≈87–105、紅血球 B≥117）。

**解法（兩部分）**：
1. **白血球核心驗證**：`detect_wbc` 框出候選後，再檢查核心區須「飽和度中位數 ≥ 100 或 B ≤ 102」，
   把淡紫紅血球團擋下（真核心 99% 通過，僅損失約 0.2% 召回率）。
2. **收緊排除用紫色遮罩**：`purple_mask_strict` 的 B 門檻 125/130 → **115**，
   只排除真正深紫的白血球/血小板，淡紫紅血球得以回到 RBC 偵測。
   （此遮罩只用於 RBC 排除／分水嶺／視覺化，不影響白血球偵測。）
→ 該圖 WBC 2→1（去除誤框）、RBC 4→11（GT 9，落在 ±30% 內）。
全資料集 WBC 精確率 **0.81 → 0.93**（FP 286→95）、RBC 召回率 **0.71 → 0.84**（多找回 2065 顆），
micro F1（IoU 0.3）**0.74 → 0.79**，血小板不受影響。

### 修正 6：一顆紅血球被切成兩顆（重複框）

診斷 test 圖 `03967e3a…`：左側一顆紅血球被框成**兩個**重疊的 RBC（其中一個是 FP），
右側貼邊一顆淡染紅血球漏抓。
**原因**：RBC 用 `RETR_LIST` 抽輪廓，環狀紅血球的「內孔」會各自成為一個輪廓——
這其實正是把黏連團塊切成單顆的隱性機制（也是召回率高的來源）；但**團塊的外輪廓**
也通過了門檻，產生一個包住好幾顆細胞的過大框，與內部的單顆框重疊，於是同一顆細胞被框兩次。

**試過但捨棄的方案**：改用 `RETR_EXTERNAL`（只留外輪廓）+ Distance Transform 峰值切團塊。
全資料集驗證**反而更差**——RBC F1 **0.78 → 0.73**、少抓 1445 顆真細胞。原因是 `RETR_EXTERNAL`
丟掉了內孔輪廓這個隱性且更有效的切團塊機制，而距離轉換峰值對**淡染細胞**達不到門檻
（如右側那顆貼邊淡染紅血球，最大距離僅 10.6 < 門檻 12.6）。故不採用。

**採用方案**：保留 `RETR_LIST`，加一道**包含抑制**——當某個「過大團塊框（邊長 > 1.5·r0）」
內部已有更細的單顆框中心，就丟掉該團塊框。如此移除重複框、又不損及團塊內的單顆偵測。
→ 該圖左側回到單一框（FP 2→1）。全資料集 RBC 精確率 **0.73 → 0.75**（移除 441 個重複 FP）、
召回率 0.84 → 0.82、F1 持平（0.7819 → 0.7824）；micro 精確率 **0.75 → 0.76**、F1 0.7894 → 0.7901。
**RBC 計數誤差（test）+14.1% → +9.3%**（重複框減少，計數更貼近 GT）。
**限制**：右側那顆貼邊淡染紅血球仍漏抓——那是低對比的半顆邊界細胞，屬另一類問題，
非團塊切分所能解決。

### 修正 7：整張偏紫的抹片——白血球框爆成整張圖、紅血球全被排除

逐張檢視 test 疊圖時發現,**6 / 126 張** RBC 召回率 < 0.5,其中 3 張近乎掛零
(如 `66c07dae…` 0/11、`7f825dd3…` 0/14)。
**原因（與修正 5 不同的路徑）**：這些抹片整體偏紫,`wbc_violet_mask` 在整張圖大面積中標,
經核心開運算 + 膨脹後**合併成一個橫跨全圖的連通塊**;而 `detect_wbc` 的幾何門檻只有下限、
**沒有上限**,於是回傳一個 `[0,0,W,H]`（整張圖）的白血球框。RBC 與血小板偵測會排除
「中心落在 WBC 框內」的候選 → 整張圖的紅血球全被排除,召回率歸零。

**關鍵觀察（資料驅動）**：對全資料集量測「大型候選（邊長 ≥ 4.5·r0）」的核心特徵後發現,
單靠 LAB-B 無法分界（真大型白血球 b_core 中位數 100、偽偏紫團 110,重疊嚴重）;
但**核心飽和度相對整張影像的高出量** `s_core − 影像 S 中位數` 乾淨分界:
真白血球核遠比整張圖飽和(中位數 +104),偽偏紫團只略高於背景(中位數 +24)。

**解法**：在 `detect_wbc` 對**過大候選（邊長 ≥ 4.5·r0）**加一道閘:
其核心須「飽和度高出影像中位數 ≥ 70」才採納,否則視為整張偏紫的染色假象而丟棄
（一般大小的白血球不受此閘影響,維持修正 4 的淡染召回）。
→ `66c07dae…` 該圖 RBC 0 → 9（整張圖白血球框消失,紅血球回來）。
全資料集 WBC 精確率 **0.93 → 0.93**（FP 95→84,移除 11 個假白血球框）、
RBC 召回率 **0.820 → 0.825**（找回 87 顆）、micro F1（IoU 0.3）**0.7901 → 0.7920**;
代價是 11 顆「大型但染色偏淡」的白血球被一併濾掉、血小板多 7 個 FP，皆屬可接受。

**試過但捨棄的方案（邊界半顆細胞）**：另想處理「貼影像邊界的半顆紅血球漏抓」
（test 有 134 顆邊界 FN）。量測後發現其中 **48 顆根本沒有對應候選**（低對比、前景訊號太弱,
非框形可救）,其餘多為一對一配對的競爭關係;改用「輪廓裁切框」取代 r0 方框後反而更差
（RBC F1 0.776 → 0.770）,故不採用——邊界漏抓主要受限於影像訊號本身,而非框的幾何。

### 修正 8：血小板誤判在白血球邊緣 + 兩個無效方向的驗證

逐張檢視時發現白血球核**邊緣**常冒出假血小板（如 `1299047e…`、`03967e3a…`）:
白血球核會把緊鄰的細胞質染色,在 WBC 框**外緣**留下顆粒狀紫色碎屑,通過了血小板規則。
**解法**：把血小板的 WBC 排除框往外加 **0.22·r0** 的 margin 再判斷中心是否落入。
→ 全資料集血小板精確率 **0.75 → 0.76**（FP 126→111,去掉 15 個邊緣假血小板,代價 4 個 TP）,
F1 持平 0.702;計數略往欠抓走（test 0.0% → −6.1%,仍在 ±30% 內）。

這個修正的另一價值是**兩個無效方向的資料驗證**（對「為什麼不那樣做」同樣重要）:

**無效方向 A — 放寬血小板門檻取回漏抓**：血小板漏抓多（召回 0.66）,主因是 area/color 門檻擋掉。
但量測候選元件發現 **雜訊 29015 個 vs 真血小板 541 個（54:1）**;放寬到 `s≥68、b≤124` 只多救
22 顆真的、卻放進 176 個雜訊（每救 1 顆付 ~8 個 FP）。**修正 3 的紋理門檻已是接近最佳的操作點**,
不能再鬆。

**無效方向 B — 削減 RBC 的過大／重複框**：原以為 RBC FP 多是過大團塊框。重新分類 4522 個 RBC FP
後發現:**79% 是 spurious（該處根本無 GT 細胞）**、18% 是 near-miss、只有 3% 是重複框。
再比對 spurious-FP 與真 RBC 的顏色／紋理特徵——**完全無法分辨**（框內飽和度中位數
TP 52.9 vs FP 52.3,任何門檻都等比例砍掉真假兩者）。這些 spurious 是 0.83 召回率的必然代價,
收緊門檻只會 1:1 換掉真細胞;**RBC 目前 (P 0.75 / R 0.83) 已在這套傳統方法的可達邊界**。

### 修正 9：框系統性偏小——校正框大小，拉高嚴格 IoU 下的分數

**觀察**：IoU 0.3 與 0.5 的分數落差偏大（micro 0.79 → 0.75),代表「框的位置大致對、但大小不夠準」,
必須把閾值設低才好看。
**量測（對配對成功的 TP）**：中心幾乎不偏（RBC 中心偏移中位數僅 0.06·r0）,但**框系統性偏小**——
預測框／GT 框寬度中位數:RBC **0.888**（小 11%）、WBC **0.928**（小 7%）、血小板 ~1.0（已準）。
原因:RBC 偵測抓的是紅血球的暗「環」輪廓,環比整顆 GT 框小;WBC 抓的是開運算後的實心核,也比 GT 小。
**解法**:
1. **RBC 框半徑 ×1.13**——把 pred/GT 比中位數從 0.888 拉回 ~1.0。掃描驗證此縮放使配對平均 IoU
   **0.70 → 0.76**、IoU≥0.5 比例 **94.5% → 96.2%**,且 IoU≥0.3 完全不掉(100%)。
2. **WBC 核心 pad 0.06 → 0.10**——把框長到 GT 大小。
3. （副作用處理）WBC 框變大會多排除血小板,故把血小板的 WBC 排除 margin 0.22 → 0.10·r0 抵銷,血小板維持原水準。
→ **IoU 0.5 micro F1 0.747 → 0.757**（WBC **0.84 → 0.87**、RBC **0.74 → 0.75**）;**IoU 0.3 持平**（0.792）;
RBC 計數誤差順帶改善 **+10.2% → +7.7%**（框變大後,團塊包覆框被「包含抑制」剔除得更乾淨）。
0.3↔0.5 的落差從 0.046 收斂到 0.034——框不再系統性偏小,嚴格閾值下的分數更接近寬鬆閾值。

### 修正 10：包含抑制過度——把肉眼可見的紅血球誤刪

逐張檢視 **val** 疊圖（只畫漏抓的 GT 框）時發現,部分**肉眼清楚可見**的紅血球被漏抓,
且常成簇出現在密集／相鄰細胞區。
**原因**：修正 6 的「包含抑制」是「若團塊框內含有更細的單顆框中心,就丟掉團塊框」。
但這個判斷用的是**完整的團塊框**;一顆只是**緊貼團塊邊緣**的獨立細胞,其中心也會落在框內,
於是把那個團塊框（可能涵蓋了別的、沒有單顆框的細胞）一起刪掉——該細胞就消失了。
修正 9 把框放大 ×1.13 後,團塊框更大、誤刪更嚴重（IoU 0.3 下少了約 175 顆真細胞）。
**解法**：包含抑制改用**縮到 80% 的團塊框**判斷——要求單顆框中心落在團塊框的**中央 80%**
才算重複。如此既保留修正 6 對「真重複框」的去除,又不再因邊緣相鄰而誤刪。
→ 全資料集找回 **+175 顆 RBC TP**（IoU 0.3,代價 +81 FP）;RBC F1 **0.784 → 0.788**、
micro F1 **0.792 → 0.796**（IoU 0.3）／ **0.757 → 0.762**（IoU 0.5）。

### 修正 11：相連紅血球切不開——Watershed（加分項）與 Hough 圓偵測

**問題定性**：再細查 val 的紅血球漏抓,發現 **706 / 949 顆（74%）是「貼著鄰居」的**,
只有 243 顆孤立。放大檢視二值圖,失效有兩種:相鄰細胞的暗膜**融合**成一團（內孔消失）、
或細胞膜對比不均只抓到**一段弧**（湊不成封閉輪廓）。現行 `RETR_LIST` 內孔機制對這兩種都失效。

**方向 B（Watershed + Distance Transform，加分項）**：先修好基礎——原本的前景遮罩用
「顏色／亮度」(`(S>15)&(V<252)|(L<246)`),在偏色抹片上會**淹掉全圖 ~94%**,距離轉換退化成
單一峰值（視覺化等於沒作用）。改用 **fill-holes 前景**（把暗膜環的洞填實成實心圓盤,
`rbc_filled_foreground`）後覆蓋率降到 ~50%,相連細胞形成多瓣團塊、每顆一個距離峰值,
watershed 才能沿谷底切開（App 以紅線疊圖呈現）。**但量測為計數補充時是淨損失**:
召回雖升（+610 顆）卻多了 **+2850 個 FP**（每救 1 顆付 ~4.7 個假的）,RBC 精確率 0.75→0.66、
計數誤差 +9%→**+30%（超出 ±30%）**。原因正是修正 8 證過的——spurious RBC 與真細胞在顏色/紋理
分不開,blob 先驗不足以分辨。**故 watershed 保留為加分項的視覺化與選用工具,預設不納入計數。**

**方向 A（Hough 圓偵測）→ 採用**：RBC 是半徑極一致的圓,`cv2.HoughCircles`（半徑鎖 [0.75,1.20]·r0）
用**累積投票**,即使只有一段弧或彼此重疊也能投出整個圓——正好打中上述兩種失效。關鍵是
「圓形 + 固定半徑」這個**幾何先驗比 watershed 的 blob 強**,能把真細胞和雜訊分開。當補充候選
（additive,排除 WBC/紫色區、與現有框去重）。全資料集驗證:Hough 找回 **+645 顆 RBC TP、
僅 +350 個 FP**（每救 1 顆付 ~0.5 個假的,遠優於 watershed）。
→ RBC 召回率 **0.825 → 0.864**、精確率僅 0.755 → 0.749、F1 **0.788 → 0.803**（IoU 0.3）／
**0.757 → 0.775**（IoU 0.5）;micro F1 **0.796 → 0.808**（IoU 0.3,進位 0.81）。計數 RBC 升到 +15.4%
（仍在 ±30% 內）。WBC、血小板完全不受影響。

**這對照本身就是結論**:同樣要提升相連細胞的召回,watershed（blob 先驗）會等比例帶進假陽性,
Hough（圓形 + 固定半徑先驗）才真正分得開——先驗的選擇決定成敗。

## 六、Watershed + Distance Transform（重疊細胞，加分項）

RBC 常緊密重疊。關鍵在**前景**：紅血球是暗膜「環」,直接用顏色/亮度當前景會在偏色抹片上
淹掉全圖 ~94%,距離轉換因此退化。改用 **fill-holes 前景**（`rbc_filled_foreground`：
把暗膜環的洞填實成實心圓盤）後覆蓋率降到 ~50%,相連細胞才形成多瓣團塊。
`watershed_visualization()` 以 Distance Transform 局部峰值（`skimage.peak_local_max`）為
marker、`skimage.watershed` 沿距離谷底切開黏合邊界,於 App 以紅線疊圖 + 黃點（峰值）顯示。

**設計決策**：watershed 候選可選擇性補進 RBC 計數（App 勾選框,`use_watershed`）,但**預設關閉**——
量測顯示純 watershed 補計數雖提升召回,卻帶進約 4.7 倍的假陽性（精確率 0.75→0.66、計數 +30% 超標,
見修正 11）。故 watershed 在此**作為重疊細胞的視覺化展示**,實際計數的相連細胞召回改由 **Hough 圓偵測**
（`hough_rbc_candidates`,修正 11）負責——圓形 + 固定半徑的先驗比 blob 先驗更能分辨真假細胞。

## 七、困難案例與限制

| 案例 | 困難點 | 緩解方式 |
|------|--------|----------|
| RBC 重疊 | 輪廓合併成大塊，純面積計數會低估 | Distance Transform + Watershed 視覺化與選用補充 |
| WBC 核附近的淡紫 | 大幅膨脹會把零散淡紫併成整張圖的框 | 開運算先取實心核，再小幅膨脹只合併同核分葉 |
| 血小板像 RBC 邊緣碎片 | 小紫點極多，顏色/大小無法分辨 | 改用顆粒紋理（`S_std`/`gray_std`）作主判據 |
| 染色變異 | 固定門檻可能漏淡染或誤收雜質 | 同時用 HSV 與 LAB 規則，並透過 UI 參數調整 |
